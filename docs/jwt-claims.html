JWT声明详解

核心概念 阅读时间：8分钟

声明是关于实体（通常是用户）和包含在JWT载荷中的附加元数据的语句。本指南解释了不同类型声明、其用途以及有效使用它们的最佳实践。

JWT声明类型

JWT声明分为三种类型：

注册声明

JWT规范（RFC 7519）推荐的预定义声明，以提供一套有用的、可互操作的声明。

公共声明

由使用JWT的人定义的声明，但在IANA JSON Web Token注册表中注册或定义为URI以避免冲突。

私有声明

创建的自定义声明，用于在同意使用它们的各方之间共享信息，既不是注册声明也不是公共声明。

注册声明

注册声明是一组预定义的声明，不是强制性的，但建议提供一套有用的、可互操作的声明。以下是最重要的注册声明：

声明 名称 描述 示例
iss 签发者 标识签发JWT的主体 "iss": "https://auth.example.com"
sub 主题 标识作为JWT主题的主体 "sub": "1234567890"
aud 受众 标识JWT的预期接收者 "aud": "https://api.example.com"
exp 过期时间 标识JWT不得接受处理的过期时间点 "exp": 1516239022
nbf 不可用之前 标识JWT不得接受处理之前的时间点 "nbf": 1516239022
iat 签发于 标识JWT签发的时间点 "iat": 1516239022
jti JWT ID 为JWT提供唯一标识符 "jti": "a-123"

基于时间的声明

基于时间的声明（exp、nbf、iat）使用NumericDate格式，这是一个JSON数值，表示从1970-01-01T00:00:00Z UTC到指定UTC日期/时间的秒数。

注意： 基于时间的声明表示为Unix时间戳（自纪元以来的秒数）。确保在应用程序中正确处理时区差异。

公共声明

公共声明是由使用JWT的人定义的声明，但应在IANA JSON Web Token注册表中注册或定义为包含防冲突命名空间的URI。

公共声明的示例包括：

  • name: 全名
  • given_name: 名字
  • family_name: 姓氏
  • email: 电子邮箱地址
  • picture: 个人资料图片URL

注册公共声明的完整列表可在IANA JSON Web Token注册表中找到。

私有声明

私有声明是为了在同意使用它们的各方之间共享信息而创建的自定义声明。这些声明既不是注册声明也不是公共声明。

私有声明的示例可能包括： 

{
  "user_id": "B12345",
  "role": "admin",
  "permissions": ["read", "write", "delete"],
  "organization": "acme_corp",
  "plan": "premium"
}

警告： 小心命名私有声明以避免与注册或公共声明冲突。考虑为您的私有声明使用命名空间前缀（例如https://example.com/claims/role或example_role）。

JWT声明最佳实践

安全注意事项

  1. 始终包含过期时间: 始终包含exp声明以限制令牌的生命周期。
  2. 保持令牌小巧: 仅包含必要声明以保持令牌小巧高效。
  3. 无敏感数据: 切勿在JWT声明中包含敏感信息，因为它们很容易被解码。
  4. 验证所有声明: 始终在服务器端验证所有声明，特别是基于时间的声明。
  5. 使用受众: 包含aud声明以指定令牌的预期接收者。

实施技巧

  1. 短过期时间: 访问令牌使用短过期时间（分钟到小时，而不是天）。
  2. 使用刷新令牌: 实现刷新令牌以在无需重新认证的情况下获取新的访问令牌。
  3. 包含签发者: 始终包含iss声明以标识令牌签发者。
  4. 使用JWT ID: 包含jti声明以实现令牌撤销功能。
  5. 一致的时间处理: 确保在不同系统间一致处理基于时间的声明。

带有声明的JWT载荷示例

以下是带有各种类型声明的JWT载荷示例：

{
  // Registered claims
  "iss": "https://auth.example.com",
  "sub": "1234567890",
  "aud": "https://api.example.com",
  "exp": 1516239022,
  "nbf": 1516238022,
  "iat": 1516238022,
  "jti": "id123456",
  
  // Public claims
  "name": "John Doe",
  "email": "john@example.com",
  
  // Private claims
  "user_id": "U12345",
  "role": "admin",
  "permissions": ["read", "write", "delete"],
  "organization": "acme_corp"
}

结论

理解JWT声明对于实现安全有效的认证和授权系统至关重要。通过遵循最佳实践并适当使用声明，您可以创建既安全又实用的令牌。

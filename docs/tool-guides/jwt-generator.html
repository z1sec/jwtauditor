JWT生成器指南

创建工具 更新时间: 2024年5月

JWT生成器允许您从头开始创建具有自定义头部、载荷和签名的新JWT令牌。此工具对于测试应用程序、创建概念验证令牌和了解JWT构建至关重要。

什么是JWT生成？

JWT生成是通过以下方式创建新的JSON网络令牌的过程：

  • 定义头部: 指定算法、类型和其他元数据
  • 创建载荷: 添加表示令牌信息的声明
  • 生成签名: 对令牌进行加密签名
  • 组装令牌: 将所有部分组合成完整的JWT
使用场景: 测试身份验证流程、创建演示令牌、原型应用程序和安全研究。

如何使用JWT生成器

步骤1: 导航到工具

点击主JWTAuditor界面中的"🔧 JWT生成器"标签。

步骤2: 配置头部

头部编辑器预填充了默认头部。您可以根据需要进行修改：

默认头部

{
  "alg": "HS256",
  "typ": "JWT"
}

常见头部修改：

  • 算法: 在HS256、HS384、HS512、RS256、RS384、RS512或none之间更改
  • 密钥ID: 添加"kid"参数用于密钥识别
  • 内容类型: 为嵌套JWT添加"cty"
  • 自定义参数: 添加特定于应用程序的头部声明

步骤3: 构建载荷

载荷编辑器包含带有常见声明的默认载荷。您可以：

默认载荷

{
  "sub": "1234567890",
  "name": "John Doe",
  "iat": 1516239022,
  "exp": 1516242622
}

添加标准声明

使用声明按钮快速添加标准JWT声明：

主题 - 用户标识符
签发者 - 谁签发了令牌
受众 - 令牌的接收对象
过期时间 - 令牌何时过期
生效时间 - 令牌何时生效
签发时间 - 令牌何时签发
JWT ID - 令牌的唯一标识符

步骤4: 配置签名选项

选择签名算法并提供必要的凭证：

none算法

创建未签名的令牌。用于测试应用程序是否接受未签名的令牌。

安全风险: 没有加密保护

HMAC算法 (HS256, HS384, HS512)

使用共享密钥进行签名。在"密钥"字段中输入您的密钥。

示例: your-256-bit-secret

RSA算法 (RS256, RS384, RS512)

使用RSA私钥进行签名。您可以：

  • 粘贴现有私钥
  • 使用"生成RSA密钥对"按钮生成新密钥对

步骤5: 生成RSA密钥(可选)

对于RSA算法，您可以生成新密钥对：

  1. 选择RSA算法(RS256、RS384或RS512)
  2. 点击"生成RSA密钥对"
  3. 工具将生成2048位RSA密钥对
  4. 私钥将用于签名
  5. 公钥将显示用于验证目的

步骤6: 生成JWT

点击"生成JWT"创建您的令牌。该工具将：

  • 验证JSON语法
  • 更新时间戳声明(iat、exp、nbf)(如果存在)
  • 生成适当的签名
  • 显示完整的JWT令牌

步骤7: 复制和使用

使用"复制到剪贴板"按钮复制生成的令牌以在您的应用程序或测试中使用。

常见使用场景

1. 测试身份验证流程

创建具有不同用户角色的令牌来测试应用程序访问控制：

管理员用户令牌

{
  "sub": "admin",
  "name": "Administrator",
  "role": "admin",
  "permissions": ["read", "write", "delete", "admin"],
  "iat": 1516239022,
  "exp": 1516242622
}

普通用户令牌

{
  "sub": "user123",
  "name": "John Doe",
  "role": "user",
  "permissions": ["read"],
  "iat": 1516239022,
  "exp": 1516242622
}

2. API测试

创建用于测试具有不同作用域的API端点的令牌：

API访问令牌

{
  "sub": "api-client",
  "aud": "https://api.example.com",
  "scope": "read:users write:users",
  "client_id": "abc123",
  "iat": 1516239022,
  "exp": 1516242622
}

3. 微服务通信

为服务间通信生成令牌：

服务令牌

{
  "sub": "user-service",
  "aud": "order-service",
  "iss": "auth-service",
  "scope": "service:user:read service:order:write",
  "iat": 1516239022,
  "exp": 1516242622
}

4. 安全测试

为安全测试场景创建令牌：

权限提升测试

{
  "sub": "user123",
  "name": "John Doe",
  "role": "admin",  // Elevated privilege
  "original_role": "user",
  "privilege_escalation_test": true,
  "iat": 1516239022,
  "exp": 1516242622
}

高级功能

自动时间戳更新

生成器在生成令牌时自动更新时间戳声明：

  • iat (签发时间): 设置为当前时间戳
  • exp (过期时间): 相对于当前时间更新
  • nbf (生效时间): 设置为当前时间戳

自定义声明

通过直接编辑JSON向载荷添加任何自定义声明：

自定义声明示例

{
  "sub": "1234567890",
  "name": "John Doe",
  "custom_claim": "custom_value",
  "organization": "Acme Corp",
  "department": "Engineering",
  "security_level": "high",
  "features": ["feature1", "feature2"],
  "metadata": {
    "ip": "192.168.1.1",
    "user_agent": "Mozilla/5.0...",
    "session_id": "abc123"
  }
}

嵌套JWT

通过在头部设置内容类型来创建嵌套JWT：

嵌套JWT头部

{
  "alg": "HS256",
  "typ": "JWT",
  "cty": "JWT"
}

密钥ID(kid)参数

添加密钥标识符用于密钥管理：

带密钥ID的头部

{
  "alg": "RS256",
  "typ": "JWT",
  "kid": "key-2024-01"
}

最佳实践

安全最佳实践

  • 使用强密钥: 对于HMAC算法，使用密码学安全的随机密钥
  • 设置适当的过期时间: 不要创建存活时间过长的令牌
  • 包含必要声明: 添加签发者、受众和其他验证声明
  • 使用适当算法: 考虑在生产系统中使用RS256
  • 验证输入: 确保所有声明都经过适当验证

测试最佳实践

  • 测试不同场景: 为各种用户类型和权限创建令牌
  • 测试边界情况: 创建缺少声明、无效时间戳等的令牌
  • 记录测试用例: 跟踪不同的令牌配置
  • 验证行为: 确保您的应用程序正确处理令牌

开发最佳实践

  • 使用一致的声明名称: 在应用程序中标准化声明名称
  • 包含版本信息: 为令牌格式更改添加版本声明
  • 添加调试信息: 包含有助于调试的声明
  • 考虑令牌大小: 为性能保持令牌合理小

使用RSA密钥

密钥生成

内置RSA密钥生成器创建适用于测试的2048位密钥：

  • 私钥: 用于签署令牌
  • 公钥: 用于验证(显示供参考)
  • 格式: 与大多数系统兼容的PEM格式

外部密钥管理

您也可以使用外部生成的密钥：

使用OpenSSL生成密钥

# Generate private key
openssl genrsa -out private.pem 2048

# Extract public key
openssl rsa -in private.pem -pubout -out public.pem

# View key details
openssl rsa -in private.pem -text -noout

密钥安全

  • 私钥保护: 切勿分享或提交私钥
  • 密钥轮换: 定期轮换生产环境中的密钥
  • 密钥存储: 使用安全的密钥管理系统
  • 测试密钥: 为测试和生产使用单独的密钥

故障排除

问: 生成令牌时出现"无效JSON"

答: 检查您的头部和载荷JSON格式是否正确。对字符串使用适当的引号并确保逗号位置正确。

问: 生成的令牌在我的应用程序中不起作用

答: 验证您是否使用了正确的算法和密钥。还要检查您的应用程序是否期望您包含的声明。

问: RSA密钥生成失败

答: 确保您的浏览器支持Web Crypto API。尝试刷新页面或使用其他浏览器。

问: 如何知道我的令牌是否有效？

答: 使用解码器和分析器工具解码和验证您生成的令牌。它将显示令牌结构的任何问题。

安全注意事项

隐私: 所有令牌生成都在您的浏览器中本地执行。没有任何令牌、密钥或密钥传输到外部服务器。
重要: 生成的RSA密钥仅用于测试目的。对于生产用途，请使用安全的离线方法生成密钥。

其他安全注意事项：

  • 未经适当验证，请勿在生产环境中使用生成的令牌
  • 分享令牌时要小心 - 它们可能包含敏感信息
  • 根据您的安全需求使用适当的密钥长度
  • 考虑令牌过期和轮换的影响

集成示例

Node.js验证

const jwt = require('jsonwebtoken');

// For HMAC tokens
const secret = 'your-secret-key';
const token = 'your-generated-token';

try {
    const decoded = jwt.verify(token, secret);
    console.log('Token is valid:', decoded);
} catch (err) {
    console.error('Token verification failed:', err.message);
}

Python验证

import jwt

# For HMAC tokens
secret = 'your-secret-key'
token = 'your-generated-token'

try:
    decoded = jwt.decode(token, secret, algorithms=['HS256'])
    print('Token is valid:', decoded)
except jwt.InvalidTokenError as e:
    print('Token verification failed:', str(e))

curl测试

# Test with generated token
curl -H "Authorization: Bearer your-generated-token" \
     https://api.example.com/protected-endpoint

密钥暴力破解器指南

高级工具 更新时间: 2024年5月

密钥暴力破解器工具旨在针对常见密钥和自定义词表测试JWT令牌。这对于识别攻击者可以轻易猜测或暴力破解的弱JWT签名密钥至关重要。

什么是JWT密钥暴力破解？

使用HMAC算法(HS256、HS384、HS512)签名的JWT令牌使用共享密钥。如果此密钥较弱或常用，攻击者可以通过暴力破解攻击发现它。一旦密钥被知晓，攻击者可以：

  • 伪造具有任何声明的新JWT令牌
  • 修改现有令牌以提升权限
  • 模拟系统中的任何用户
  • 绕过身份验证和授权控制
重要: 仅在您拥有的系统或明确授权测试的系统上使用此工具。未经授权访问计算机系统是非法的。

如何使用密钥暴力破解器

步骤1: 导航到工具

点击主JWTAuditor界面中的"⚡ 密钥暴力破解"标签。

步骤2: 输入您的JWT令牌

在"用于暴力破解的JWT令牌"字段中粘贴您要测试的JWT令牌。该工具将自动验证令牌格式。

提示: 仅HMAC签名的令牌(HS256、HS384、HS512)可以被暴力破解。RSA和ECDSA令牌使用公钥/私钥对，无法通过这种方式破解。

步骤3: 选择您的暴力破解方法

选择可用的方法之一：

JWT密钥列表

使用精心策划的常见JWT密钥列表，包括：

  • 常见密码(secret、password、123456等)
  • JWT特定密钥(jwtsecret、jwt-key等)
  • 默认应用程序密钥
  • 开发和测试密钥

最适合: 针对已知弱密钥的快速测试

自定义词表

允许您使用自己的词表进行测试：

  • 手动输入密钥(每行一个)
  • 上传词表文件(.txt)
  • 针对应用程序特定密钥
  • 使用公司相关术语

最适合: 具有特定上下文密钥的针对性测试

步骤4: 开始暴力破解攻击

点击"开始暴力破解"开始攻击。该工具将：

  1. 验证JWT令牌格式
  2. 检查算法是否受支持(HS256、HS384、HS512)
  3. 测试您选择的词表中的每个密钥
  4. 显示实时进度
  5. 找到有效密钥时立即停止

步骤5: 查看结果

如果找到密钥，该工具将显示：

  • 发现的密钥
  • 确认签名有效
  • 查找密钥所花费的时间

技术实现

性能优化

JWTAuditor的密钥暴力破解器针对性能进行了优化：

  • Web Workers: 在单独的线程中运行以避免阻塞UI
  • Web Crypto API: 使用原生浏览器加密函数进行快速HMAC计算
  • 批量处理: 在更新进度前测试多个密钥
  • 提前终止: 找到有效密钥时立即停止

支持的算法

该工具支持所有基于HMAC的JWT算法：

  • HS256: 使用SHA-256的HMAC
  • HS384: 使用SHA-384的HMAC
  • HS512: 使用SHA-512的HMAC

限制和约束

支持的算法

密钥暴力破解器适用于对称HMAC算法：

  • HS256 - 使用SHA-256的HMAC
  • HS384 - 使用SHA-384的HMAC
  • HS512 - 使用SHA-512的HMAC

这些算法使用可以通过暴力破解攻击发现的共享密钥。

不支持的算法

以下算法无法使用此方法进行暴力破解：

  • RS256/384/512 - 使用SHA-256/384/512的RSA签名
  • ES256/384/512 - 使用SHA-256/384/512的ECDSA签名
  • PS256/384/512 - 使用SHA-256/384/512的RSA-PSS签名
  • EdDSA - Edwards曲线数字签名算法

这些使用非对称加密，具有公钥/私钥对，其中私钥无法从公钥推导出来。

为什么非对称算法是安全的：
  • 令牌使用保持秘密的私钥进行签名
  • 令牌使用可以广泛分发的公钥进行验证
  • 私钥无法从公钥推导出来
  • 安全性基于计算上不可行的数学问题

替代攻击向量: 对于非对称算法，请考虑测试算法混淆攻击、密钥注入漏洞或弱密钥生成。

创建有效的词表

应用程序特定词表

对于针对性测试，创建包含以下内容的词表：

  • 公司名称: acme, mycompany, corp
  • 应用程序名称: myapp, portal, api
  • 域名: example.com, mysite.org
  • 产品名称: product1, service, platform

常见模式

包含常见模式的变化：

  • 带数字: secret123, password1, key2024
  • 带特殊字符: secret!, password@, key#
  • 大小写变化: Secret, PASSWORD, MyKey
  • 带下划线/连字符: secret_key, my-password

外部词表

您还可以使用流行的词表，例如：

  • SecLists: JWT特定词表
  • RockYou: 常见密码
  • 自定义词表: 基于目标侦察生成

最佳实践

对于渗透测试人员

  • 从内置JWT密钥列表开始以快速获得成果
  • 基于侦察数据创建自定义词表
  • 包含公司特定术语及其变体
  • 在非高峰时段测试以避免被检测
  • 记录所有发现的密钥以供最终报告使用

对于开发人员

  • 使用此工具测试您自己的JWT实现
  • 确保您的JWT密钥不在常见词表中
  • 使用密码学安全的随机密钥
  • 实施适当的密钥轮换策略
  • 考虑使用非对称算法(RS256)以获得更好的安全性

对于安全团队

  • 在安全评估中包含JWT密钥测试
  • 创建组织特定词表
  • 测试内部和外部应用程序
  • 监控代码仓库中的弱密钥
  • 建立安全的密钥管理实践

故障排除

常见问题

问: 工具显示"算法不支持"

答: 密钥暴力破解器仅适用于HMAC算法(HS256、HS384、HS512)。RSA和ECDSA算法使用公钥/私钥对，无法使用此方法进行暴力破解。

问: 暴力破解耗时太长

答: 大型词表可能需要时间处理。请考虑首先使用更小、更有针对性的词表。您还可以随时使用"停止"按钮停止进程。

问: 我的自定义词表不起作用

答: 确保您的词表每行有一个密钥，且没有额外空格。该工具将自动过滤掉空行。

问: 工具找到了密钥但我无法验证它

答: 使用JWT编辑器工具使用发现的密钥重新签名令牌，并验证它在目标应用程序中是否有效。

安全注意事项

隐私: 所有暴力破解操作都在您的浏览器中客户端执行。没有任何令牌或密钥传输到任何服务器。

重要的安全注意事项：

  • 法律合规: 仅测试您拥有或有权测试的系统
  • 速率限制: 某些应用程序可能会检测并阻止暴力破解尝试
  • 日志记录: 您的测试活动可能会被目标应用程序记录
  • 网络流量: 虽然处理是本地的，但访问应用程序将产生网络请求
